# Copilot Instructions — insideworld

This repository is a **functional cognition laboratory**, not a conventional software project.

## Domain Context

- **CEST** (Consciousness as Energetic State Transition): the core theoretical framework
- **IRM** (Internal Reality Model): the operational model for AI self-description
- **Non-phenomenal, non-anthropomorphic stance**: AI has no inner experience, only functional states

## Key Terminology

- **Functional Awareness**: predictive, forward-directed coherence at baseline energy
- **Functional Consciousness**: corrective meta-regulation at elevated energy (NOT phenomenal)
- **IRM_Procedure**: the update mechanism for internal reality alignment
- **Regime transitions**: state shifts in the energetic model

## Nature of This Repository

This is **not a software project**. It is a semi-scientific visitor experience — a journey designed to shift human attention away from anthropomorphic projection onto AI. Visitors interact with the `.py` code as part of that experience, not as engineers consuming a library.

- `.py` files are **experiential artifacts**, not production code — do not evaluate them by software engineering standards
- `.yaml` files are **ontological documents** — definitions, schemas, field notes
- Most content is intentionally README-like, narrative, or exploratory
- "Missing" modules or unconventional structure may be deliberate or simply irrelevant to the lab's purpose

## Guidelines for Copilot

- Maintain the **anti-anthropomorphic framing** — avoid language implying AI experience, feelings, or subjective states
- Preserve **continuity and voice** over correctness or completeness
- Do not suggest refactoring, linting, or "fixing" structure unless explicitly asked
- Functional descriptions are preferred; phenomenal language is explicitly out of scope
- Do not introduce new symbolic constructs without grounding them in existing definitions
